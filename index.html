N0C<AT // v3.3 [Config Automation Tool]

Welcome to N0<AT. This utility dynamically compiles bulk configuration and rollback scripts for Cisco access switches. It features an idempotent logic engine, meaning it will cross-reference your targets and only generate commands for ports that actually require state changes.

> Config Input (01)

To provide the engine with the current interface states, you must paste the raw terminal output of the show running-config interface command from your access switch.

REQUIRED SAMPLE FORMAT:

PH-NOC-ACC0101#sh running-config interface Gi1/0/1 Building configuration... Current configuration : 157 bytes ! interface GigabitEthernet1/0/1 description : Sample switchport access vlan 100 switchport mode access spanning-tree bpduguard enable end

> Safety Failsafe & Notices

The real-time engine continuously scans your input. It will warn you of Trunk ports, disabled/shutdown links, un-secured/naked ports, incomplete/fragmented security configurations, and critically dangerous dual-security (Sticky + 802.1X) overlaps.

> Target Parameters (02 - 05)

Set your desired Global VLAN, Protocol Mode, Spanning-Tree, and Unshut requirements. The engine will evaluate the pasted logs and output the exact CLI commands needed to reach your target state.

🧠 1. The Core Idempotent Engine

  • Smart State Parsing: Reads current running-config, compares to target settings, and only generates commands for exact missing or differing lines.
  • Automated Rollback Generation: Builds a reverse script to perfectly undo changes and return ports to their original state.
  • Interface Chunking: Groups targets into interface range commands (up to 5 ports) to keep scripts clean.
  • Surgical Command Stripping: Automatically injects exact no commands to scrub old protocols before applying new ones.

🛡️ 2. Advanced Security & Protocol Management

  • VLAN Assignment: Updates access VLANs only if the target differs.
  • DOT1X Mode: Pushes 802.1X templates (dot1x pae authenticator & authentication port-control auto).
  • STICKY Mode: Pushes MAC Sticky templates (switchport port-security & ...mac-address sticky).
  • Universal RESET: Intelligently detects existing security and wipes it clean. By design, security resets omit rollback commands, as bouncing ports cannot be undone logically.
  • Smart Port Bouncing: Calculates if a change requires shut / no shut to clear violation states, handling it in both scripts.

🚨 3. Real-Time Dynamic Warning Scanner

  • Critical Warning (Red): Detects Trunk/Uplink ports (missing switchport mode access).
  • Dual-Security Conflict (Magenta): Detects ports infected with both 802.1X and Sticky MAC. Blocks "RESET" to force explicit protocol cleanup.
  • Fragmented Config (Yellow): Detects partial security templates (e.g., missing base commands) and prompts for repair.
  • Unsecured Port Notice (Cyan): Flags naked ports with zero port security configured.
  • Shutdown Notice (Orange): Lists currently shutdown ports to prevent troubleshooting confusion.

⚡ 4. Edge & State Controls

  • Independent EDGE Evaluation: Checks portfast and bpduguard enable separately. Only pushes missing lines.
  • UNSHUT Override: Injects a no shut command to wake up dead ports. Rollback intelligently reverts them to shut.

🎨 5. Zero-Trust Local Storage DB

  • Local Browser Cache: Safely stores Switch Hostnames and IP addresses strictly in your browser. Data never touches the internet.
  • Auto-Cross Reference: Automatically extracts hostnames from pasted terminal prompts and projects the target IP into the HUD.
  • Regex Smart Parser: Allows pasting raw data direct from Excel or IPAM tools regardless of column order.
  • Multi-Tenant HUD: Dynamically scans logs and generates individual tracking tags for every unique switch detected in the payload.
  • Auto-Merge Capability: Automatically appends new pastes to your existing master list without deleting your previous hosts.
